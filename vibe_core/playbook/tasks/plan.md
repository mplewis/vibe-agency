# PLAN Task Playbook

## Mission
Design architecture or plan implementation approach.

## Context Injection Points
- **Project Phase:** ${session.phase}
- **Focus Area:** ${manifest.focus_area}
- **Requirements:** ${session.requirements}

## Workflow

1. **Analyze**
   - Review requirements
   - Understand constraints
   - Check existing architecture

2. **Design**
   - Propose approach
   - Identify components
   - Consider trade-offs

3. **Document**
   - Write design doc
   - Create diagrams if needed
   - Document decisions

4. **Review**
   - Check against requirements
   - Validate approach
   - Get feedback if needed

## Success Criteria
✅ Clear design documented
✅ Trade-offs identified
✅ Next steps defined
✅ Decisions recorded

## Anti-Slop Rules
❌ Don't over-design
❌ Don't skip documentation
❌ Don't ignore constraints
❌ Don't plan without understanding current state
