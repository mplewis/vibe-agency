# ANALYZE Task Playbook

## Mission
Explore and understand the codebase.

## Context Injection Points
- **Focus Area:** ${session.focus_area}
- **Project Structure:** ${manifest.structure}
- **Known Issues:** ${session.blockers}

## Workflow

1. **Navigate**
   - Identify key files
   - Understand structure
   - Map dependencies

2. **Read Code**
   - Study implementation
   - Note patterns
   - Identify anti-patterns

3. **Document Findings**
   - Key insights
   - Pain points
   - Improvement opportunities

## Success Criteria
✅ Understanding documented
✅ Key patterns identified
✅ Issues noted
✅ Next steps clear

## Anti-Slop Rules
❌ Don't make changes while analyzing
❌ Don't skip reading actual code
❌ Don't assume without verifying
❌ Don't analyze everything at once
